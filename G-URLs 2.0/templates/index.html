About G-URLs 2.0

Multi-Layer Behavioural Detection Architecture for Malicious Domain and Web Traffic Analysis

Academic Context

Masters dissertation artefact in Cybersecurity implementing a multi-layer behavioural detection architecture evaluated across four datasets comprising over 2.3 million labelled samples.

Four Detection Layers

DNS infrastructure analysis, TLS certificate metadata, network flow behaviour, and URL lexical structure — each catching different threat subsets to provide complementary coverage.

ML Enhancement

XGBoost models trained on all four datasets provide predictions alongside rule-based verdicts. Rule-based F1: DNS 0.675, TLS 0.359, Network 0.828, URL 0.978. XGBoost F1: DNS 0.929, TLS 0.773, Network 0.998, URL 1.000.

VirusTotal Integration

Results cross-referenced against 70+ antivirus engines providing direct comparison between behavioural detection and reputation-based industry detection — validating the Phase 4 baseline findings.

Detection Modules

Module A — DNS Behaviour (F1: 0.675, Recall: 0.903)

Live DNS lookup analysing TTL, MX records, nameserver patterns, domain age, and DNSSEC. 18 weighted rules calibrated on 530,468 Zenodo CESNET domains. High recall design surfaces 90.3% of malicious domains for review.

Module B — TLS Metadata (F1: 0.359)

Live TLS handshake reading certificate issuer, validity, CT log presence, and chain length. Low standalone F1 reflects APT actors deliberately using legitimate certificates — motivating multi-layer combination.

Module C — Network Behaviour (F1: 0.828)

Network connectivity probing for HTTP/ HTTPS availability, redirect behaviour, and response characteristics. Full packet-level analysis requires network monitoring infrastructure deployment.

Module D — URL Lexical (F1: 0.978)

URL string analysis — length, similarity index, digit ratio, HTTPS, special characters. Strongest standalone module. Expert rules achieve F1 0.978 vs ML perfect 1.000 — 0.022 gap demonstrating near-optimal rule design.
